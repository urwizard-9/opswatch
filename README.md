# OpsWatch

**배포 서버 통신 상태 모니터링 및 장애 이력관리 DevOps 시스템**

인공지능파이프라인 중간프로젝트(26-1)

> 여러 서버/API의 통신 상태를 점검하여 UP/SLOW/DOWN을 판별하고,  
> 점검 이력·장애 이력·로그·메트릭을 관리하는 FastAPI 기반 운영 모니터링 시스템

---

## 커밋 메시지 규칙

본 프로젝트는 다음과 같은 커밋 메시지 규칙을 사용합니다.

```
<type>: <설명>
```

| Type | 의미 | 예시 |
|---|---|---|
| `feat` | 새로운 기능 추가 | `feat: 서버 등록 API 구현` |
| `fix` | 버그 수정 | `fix: slow 응답 판별 기준 수정` |
| `test` | 테스트 추가 또는 수정 | `test: 서버 등록 API 테스트 추가` |
| `ci` | CI/CD 설정 변경 | `ci: GitHub Actions 테스트 워크플로우 추가` |
| `chore` | 빌드, 도구, 설정 등 기능 외 변경 | `chore: Dockerfile 추가` |
| `docs` | 문서 작성 또는 수정 | `docs: README 실행 방법 작성` |
| `refactor` | 기능 변경 없는 코드 구조 개선 | `refactor: 상태 점검 로직 분리` |
| `style` | 코드 스타일 변경 (포매팅, 세미콜론 등) | `style: ruff 린트 경고 수정` |
